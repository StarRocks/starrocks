```java
@Override
public Statistics getTableStatistics(OptimizerContext session,
                                     Table table,
                                     Map<ColumnRefOperator, Column> columns,
                                     List<PartitionKey> partitionKeys,
                                     ScalarOperator predicate,
                                     long limit,
                                     TvrVersionRange tableVersionRange) {
    Statistics.Builder builder = Statistics.builder();

    // Default: unknown stats for all columns
    for (ColumnRefOperator colRef : columns.keySet()) {
        builder.addColumnStatistic(colRef, ColumnStatistic.unknown());
    }

    // Try to get row count from remote via COUNT(*)
    ADBCTable adbcTable = (ADBCTable) table;
    try {
        long rowCount = getRemoteRowCount(adbcTable);
        builder.setOutputRowCount(rowCount);
    } catch (Exception e) {
        LOG.warn("Failed to get row count for ADBC table {}: {}", adbcTable.getName(), e.getMessage());
        builder.setOutputRowCount(1);  // Fallback
    }

    return builder.build();
}
```

Add private helper `getRemoteRowCount(ADBCTable table)`:
- Build SQL: `SELECT COUNT(*) FROM "dbName"."tableName"` (use double-quote identifier quoting, matching the identifier quote pattern from ADBCScanNode)
- Open connection via `adbcDatabase.connect()`
- Create statement, set SQL, execute query
- Read first batch, get BigIntVector at index 0, return value
- Use try-with-resources for connection, statement, reader
- On any failure, let exception propagate to caller (caught in getTableStatistics)

Do NOT implement per-column stats collection in this plan (per CONTEXT.md, stats feed via getTableStatistics is row-count focused; ANALYZE TABLE handles per-column stats through the existing scan path). Set all columns to `ColumnStatistic.unknown()`.
