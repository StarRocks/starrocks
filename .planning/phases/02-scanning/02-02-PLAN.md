Create ADBCParallelReader that implements multi-endpoint parallel reading:

```cpp
#pragma once

#include <arrow-adbc/adbc.h>
#include <arrow/record_batch.h>
#include <atomic>
#include <condition_variable>
#include <memory>
#include <mutex>
#include <queue>
#include <thread>
#include <vector>

#include "common/status.h"

namespace starrocks {

// Thread-safe bounded queue for RecordBatch transfer between reader threads and consumer
class RecordBatchQueue {
public:
    explicit RecordBatchQueue(size_t capacity = 16);
    // Push a batch (blocks if full). nullptr sentinel signals completion.
    void push(std::shared_ptr<arrow::RecordBatch> batch);
    // Pop a batch (blocks if empty). Returns nullptr when all producers done.
    std::shared_ptr<arrow::RecordBatch> pop();
    // Signal that a producer is done (when all producers signal done and queue empty, pop returns nullptr)
    void producer_done();
    void set_error(Status status);
    Status error() const;

private:
    std::queue<std::shared_ptr<arrow::RecordBatch>> _queue;
    mutable std::mutex _mutex;
    std::condition_variable _not_full;
    std::condition_variable _not_empty;
    size_t _capacity;
    std::atomic<int> _active_producers{0};
    Status _error;
    bool _has_error = false;
};

// Reads multiple ADBC partitions in parallel, each on its own thread
class ADBCParallelReader {
public:
    ADBCParallelReader(AdbcDatabase* database, const AdbcPartitions& partitions,
                       size_t num_threads);
    ~ADBCParallelReader();

    Status start();
    // Returns next batch from any endpoint. Sets *eos=true when all done.
    Status get_next(std::shared_ptr<arrow::RecordBatch>* batch, bool* eos);
    void close();

private:
    void _reader_thread(const uint8_t* partition_data, size_t partition_length);

    AdbcDatabase* _database;  // borrowed, not owned
    std::vector<std::pair<const uint8_t*, size_t>> _partitions;
    size_t _num_threads;
    RecordBatchQueue _queue;
    std::vector<std::thread> _threads;
    bool _started = false;
    bool _closed = false;
};

} // namespace starrocks
```

Implementation notes for adbc_parallel_reader.cpp:
- RecordBatchQueue: standard bounded blocking queue with mutex + condition_variable. Capacity defaults to 16 batches.
- `_reader_thread(partition_data, partition_length)`:
  1. Open a NEW AdbcConnection per thread (from shared _database)
  2. Call `AdbcConnectionReadPartition(&conn, partition_data, partition_length, &stream, &error)`
  3. Import stream via `arrow::ImportRecordBatchReader(&stream)`
  4. Loop: ReadNext batch, push to queue
  5. On completion or error: call `_queue.producer_done()`, release connection
- `start()`: Set `_active_producers = _partitions.size()`, spawn one thread per partition (capped at num_threads; if more partitions than threads, use a work-stealing approach or sequential assignment)
- `get_next()`: Pop from queue. If nullptr and no error, set eos=true. If error, return error.
- `close()`: Join all threads. Release queue.
- Thread safety: Each thread has its OWN AdbcConnection. Only the queue is shared. Careful with closures capturing shared state.

**adbc_scanner.cpp:**

1. Define `RETURN_ADBC_NOT_OK(status_code, adbc_error)` macro that:
   - Checks if `status_code != ADBC_STATUS_OK`
   - Extracts error.message (or "Unknown ADBC error" if null)
   - Calls `error.release(&error)` if release function is set
   - Returns `Status::InternalError("ADBC error: " + msg)`

2. **Constructor:** Store all parameters. Set `_max_chunk_size` from RuntimeState config (state->chunk_size()) in open(), or default 4096.

3. **open():**
   - Record start time: `auto start = MonotonicMillis();`
   - Call `_init_adbc()` to set up database/connection/statement
   - Record `_connect_time_ms = MonotonicMillis() - start;`
   - Try parallel reading first: call `_try_parallel_read(state)`
   - If parallel fails (returns non-OK), fall back to `_fallback_single_stream_read()`
   - Set `_opened = true`

4. **_init_adbc():**
   - `AdbcDatabaseNew(&_database, &error)` + RETURN_ADBC_NOT_OK
   - `AdbcDatabaseSetOption(&_database, "driver", _driver.c_str(), &error)` -- use the driver name directly (e.g., "adbc_driver_flightsql")
   - `AdbcDatabaseSetOption(&_database, "uri", _uri.c_str(), &error)`
   - If `_username` not empty: `AdbcDatabaseSetOption(&_database, "username", ...)`
   - If `_password` not empty: `AdbcDatabaseSetOption(&_database, "password", ...)`
   - If `_token` not empty: `AdbcDatabaseSetOption(&_database, "adbc.flight.sql.authorization_header", ("Bearer " + _token).c_str(), ...)`
   - `AdbcDatabaseInit(&_database, &error)` + RETURN_ADBC_NOT_OK, set `_database_initialized = true`
   - `AdbcConnectionNew(&_connection, &error)` + RETURN_ADBC_NOT_OK
   - `AdbcConnectionInit(&_connection, &_database, &error)` + RETURN_ADBC_NOT_OK, set `_connection_initialized = true`
   - `AdbcStatementNew(&_connection, &_statement, &error)` + RETURN_ADBC_NOT_OK, set `_statement_initialized = true`
   - `AdbcStatementSetSqlQuery(&_statement, _sql.c_str(), &error)` + RETURN_ADBC_NOT_OK

5. **_try_parallel_read(state):**
   - Call `AdbcStatementExecutePartitions(&_statement, &schema, &partitions, &rows_affected, &error)`
   - If this returns non-OK (driver doesn't support partitions), return the error (caller falls back)
   - If `partitions.num_partitions <= 1`, return error to trigger single-stream fallback (not worth parallel overhead)
   - Create `_parallel_reader = std::make_unique<ADBCParallelReader>(&_database, partitions, num_threads)`
   - Thread pool size: use `std::min(partitions.num_partitions, (size_t)4)` (Claude's discretion per CONTEXT.md)
   - Call `_parallel_reader->start()`
   - Set `_use_parallel = true`
   - Release partitions memory: `partitions.release(partitions)` if release function is set

6. **_fallback_single_stream_read():**
   - Execute query: `AdbcStatementExecuteQuery(&_statement, &c_stream, &rows_affected, &error)`
   - Import stream: `auto result = arrow::ImportRecordBatchReader(&c_stream)`
   - Check result status, store in `_batch_reader`
   - Set `_use_parallel = false`

7. **get_next():**
   - If `_pending_batch` has remaining rows (from re-chunking), consume from `_pending_offset`
   - Otherwise:
     - If `_use_parallel`: call `_parallel_reader->get_next(&batch, &eos_flag)`
     - Else: call `_batch_reader->ReadNext(&batch)`. If batch is null, set eos_flag = true.
   - If eos_flag, set `*eos = true`, return OK.
   - If batch->num_rows() > `_max_chunk_size`: store as `_pending_batch`, set `_pending_offset = 0`, slice to `_max_chunk_size` rows using `batch->Slice(offset, length)`
   - Call `_convert_batch_to_chunk(sliced_batch, chunk)`
   - Update `_rows_read` and `_bytes_read` stats

8. **_convert_batch_to_chunk():**
   - Create a new Chunk with columns matching `_tuple_desc->slots()`
   - For each materialized slot in `_tuple_desc`:
     - Get arrow column from batch by index
     - Use `get_arrow_converter()` from `arrow_to_starrocks_converter.h` to get ConvertFunc
     - Call converter to fill StarRocks Column
   - Set chunk num_rows
   - Return OK

9. **close():**
   - Guard against double-close: if `_closed` return
   - If `_parallel_reader`, call `_parallel_reader->close()`
   - Release in reverse order: statement, connection, database
   - Only release if the corresponding `_initialized` flag is true
   - Use AdbcError for each release call but don't fail on release errors (log warning)
   - Set `_closed = true`

10. **Destructor:** Call close(nullptr) if not already closed.

CRITICAL PITFALLS (from research):
- Hold AdbcStatement/Connection/Database alive until RecordBatchReader is fully consumed (they are released in close(), not after execute)
- Always check ADBC_STATUS_OK on every C API call
- Use RAII-like flags (_database_initialized etc.) to ensure correct cleanup even on partial initialization failure
- The driver name passed to AdbcDatabaseSetOption must be "adbc_driver_flightsql" (the C library name), NOT "flight_sql" (the user-facing property). The FE performs this translation.
- Each parallel reader thread must open its OWN AdbcConnection (connection is NOT thread-safe)
- LIMIT correctness with parallel reading: if LIMIT was pushed down in the SQL, each endpoint returns a subset; the scanner must still respect the limit by stopping early if enough rows collected
