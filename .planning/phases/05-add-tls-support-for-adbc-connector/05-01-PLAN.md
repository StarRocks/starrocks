  **1. ADBCConnector.java -- Add TLS property constants and validation:**

  Add static final constants after PROP_URL:
  ```java
  public static final String PROP_TLS_CA_CERT_FILE = "adbc.tls.ca_cert_file";
  public static final String PROP_TLS_CLIENT_CERT_FILE = "adbc.tls.client_cert_file";
  public static final String PROP_TLS_CLIENT_KEY_FILE = "adbc.tls.client_key_file";
  public static final String PROP_TLS_VERIFY = "adbc.tls.verify";
  ```

  Add a `private static boolean isTlsUri(String uri)` method -- case-insensitive check for `grpc+tls://` prefix. Per CONTEXT.md, only `grpc+tls://` is recognized (not `grpcs://`).

  Add a `private void validateCertFile(String key)` method that checks if the property is set, and if so validates: file exists, is a regular file, is readable. Throw `StarRocksConnectorException` with clear message on failure.

  Add a `private boolean hasTlsCertProperties()` helper checking if any of the three cert file properties are set.

  In the constructor, after the existing `validate(PROP_URL)` call, add TLS validation logic:
  - Get URI from properties, call `isTlsUri(uri)`
  - If NOT TLS but `hasTlsCertProperties()`: log WARNING "TLS certificate properties provided but URI scheme is not 'grpc+tls://'. Certificates will be ignored. Use 'grpc+tls://' to enable TLS." (per CONTEXT.md decision)
  - If TLS: validate each cert file property via `validateCertFile()`. Then check mTLS pair: if only one of client_cert/client_key is set, throw StarRocksConnectorException with message "adbc.tls.client_cert_file and adbc.tls.client_key_file must both be provided for mTLS".

  **2. ADBCMetadata.java -- Add TLS options to initDatabase():**

  Add import for `org.apache.arrow.adbc.driver.flightsql.FlightSqlConnectionProperties` and `java.io.FileInputStream`.

  In `initDatabase()`, after the existing username/password block (after line 106), add TLS option logic:
  - Check if URI starts with `grpc+tls://` (case-insensitive)
  - If TLS:
    - If `adbc.tls.ca_cert_file` is set: read file as FileInputStream, call `FlightSqlConnectionProperties.TLS_ROOT_CERTS.set(params, inputStream)`. Note: driver reads stream during `open()`, so no early-close issue.
    - If both `adbc.tls.client_cert_file` and `adbc.tls.client_key_file` are set: read both as FileInputStream, call `FlightSqlConnectionProperties.MTLS_CERT_CHAIN.set(params, certStream)` and `FlightSqlConnectionProperties.MTLS_PRIVATE_KEY.set(params, keyStream)`.
    - If `adbc.tls.verify` is `"false"` (case-insensitive): call `FlightSqlConnectionProperties.TLS_SKIP_VERIFY.set(params, true)` and log WARNING "ADBC catalog '{}': TLS certificate verification DISABLED (insecure mode)".
  - If NOT TLS: do nothing (per CONTEXT.md: certs ignored with non-TLS URI)

  **3. PlanNodes.thrift -- Add TLS fields to TADBCScanNode:**

  Add fields 11-14 to TADBCScanNode struct (after field 10: adbc_token):
  ```thrift
  11: optional string adbc_tls_ca_cert_file
  12: optional string adbc_tls_client_cert_file
  13: optional string adbc_tls_client_key_file
  14: optional bool adbc_tls_verify
  ```
  Use `optional` per Thrift guidelines (never use `required`).

  **4. ADBCScanNode.java -- Add TLS fields to toThrift():**

  In `toThrift()` method, after the token block (line ~228), add:
  - Get `adbc.tls.ca_cert_file` via `getPropertyValue()`, if non-null call `adbcScanNode.setAdbc_tls_ca_cert_file()`
  - Get `adbc.tls.client_cert_file`, if non-null call `adbcScanNode.setAdbc_tls_client_cert_file()`
  - Get `adbc.tls.client_key_file`, if non-null call `adbcScanNode.setAdbc_tls_client_key_file()`
  - Get `adbc.tls.verify`, default to `"true"`. Call `adbcScanNode.setAdbc_tls_verify(!"false".equalsIgnoreCase(value))` -- default true when not set, false only when explicitly "false". Use the ADBCConnector.PROP_TLS_VERIFY constant for the property key.
