static std::string get_sqlite_driver_path() {
    // 1. Env var override
    const char* env = std::getenv("ADBC_SQLITE_DRIVER_PATH");
    if (env && std::strlen(env) > 0) return env;

    // 2. Vendored build output
    std::string vendored = "thirdparty/installed/lib64/libadbc_driver_sqlite.so";
    if (access(vendored.c_str(), R_OK) == 0) return vendored;

    // 3. Phase 1 spike install
    std::string home = std::getenv("HOME") ? std::getenv("HOME") : "";
    std::string spike = home + "/.config/adbc/drivers/sqlite_linux_amd64_v1.10.0/libadbc_driver_sqlite.so";
    if (access(spike.c_str(), R_OK) == 0) return spike;

    return ""; // test will skip if empty
}

**Test 2: ScannerInitViaDriverManager (per D-18 item 1)**
```
TEST(ADBCSmokeTest, ScannerInitViaDriverManager)
```
- Get SQLite driver path; GTEST_SKIP if not found
- Build `ADBCScanContext`: driver_url = sqlite_path, uri = ":memory:", sql = "SELECT 1"
- This test validates that `_init_adbc()` works through the driver manager.
- Since we cannot easily construct a full `TupleDescriptor` / `RuntimeState` in a unit test without the full FE pipeline, test the driver registry + ADBC database init directly:
  - Create AdbcDatabase, call AdbcDatabaseSetOption("driver", path), AdbcDatabaseInit
  - Create AdbcConnection, AdbcConnectionInit
  - Create AdbcStatement, SetSqlQuery("SELECT 1"), ExecuteQuery
  - Verify the result ArrowArrayStream has a valid schema
  - Clean up via AdbcStatementRelease, AdbcConnectionRelease, AdbcDatabaseRelease

**Test 3: EndToEndSelect (per D-18 item 2)**
```
TEST(ADBCSmokeTest, EndToEndSelect)
```
- Get SQLite driver path; GTEST_SKIP if not found
- Using raw ADBC C API (not through scanner, since scanner needs RuntimeState):
  - AdbcDatabaseNew + SetOption("driver", path) + SetOption("uri", ":memory:") + Init
  - AdbcConnectionNew + Init
  - Execute "CREATE TABLE test_t (id INTEGER, name TEXT)" via AdbcStatementNew + SetSqlQuery + ExecuteQuery
  - Execute "INSERT INTO test_t VALUES (1, 'alice'), (2, 'bob')"
  - Execute "SELECT id, name FROM test_t ORDER BY id"
  - Read ArrowArrayStream: get schema, get batches
  - Assert: got rows, first row id=1 name=alice, second row id=2 name=bob
  - Use ArrowArrayStreamHolder RAII wrapper for the result stream
  - Clean up

**Test 4: ConcurrentScan (per D-18 item 3, TEST-04)**
```
TEST(ADBCSmokeTest, ConcurrentScan)
```
- Get SQLite driver path; GTEST_SKIP if not found
- Set up one shared AdbcDatabase with driver + uri=":memory:"
- Create the test table and insert 100 rows
- Launch 8 threads, each thread:
  - Creates its own AdbcConnection (per-fragment, per BE-03)
  - Creates its own AdbcStatement
  - Executes "SELECT * FROM test_t"
  - Reads all rows from the ArrowArrayStream
  - Asserts row count == 100
  - Releases statement, connection
- Join all threads
- Assert no thread threw an exception
- Release database

Use `std::atomic<int> success_count` to track successful thread completions. Assert `success_count == 8`.
