In `thirdparty/vars.sh`, add ADBC variables directly after the ARROW_MD5SUM block (after line 253). Use this exact block:

```bash
# ADBC C++ library (Apache Arrow ADBC - for BE native data scanning)
ADBC_DOWNLOAD="https://github.com/apache/arrow-adbc/archive/refs/tags/apache-adbc-1.1.0.tar.gz"
ADBC_NAME="arrow-adbc-apache-adbc-1.1.0.tar.gz"
ADBC_SOURCE="arrow-adbc-apache-adbc-1.1.0"
ADBC_MD5SUM=""
```

The MD5SUM is intentionally empty here — the actual value must be computed at runtime. In `build-thirdparty.sh`, add a `check_md5sum` call that computes it on first download OR leave the MD5SUM check non-fatal for this entry by following how the build script handles missing checksums (check existing pattern in the script). If the build script uses `check_if_source_exist` which skips MD5 on pre-existing extracts, that pattern is fine.

In `thirdparty/build-thirdparty.sh`, add the `build_adbc()` function. Place it AFTER the `build_arrow()` function definition (order matters: ADBC depends on Arrow headers). The function builds from the `c/` subdirectory of the ADBC source (NOT the Java source):

```bash
build_adbc() {
    check_if_source_exist $ADBC_SOURCE
    cd $TP_SOURCE_DIR/$ADBC_SOURCE/c
    mkdir -p build && cd build
    rm -rf CMakeCache.txt CMakeFiles/
    ${CMAKE_CMD} \
        -DADBC_DRIVER_FLIGHTSQL=ON \
        -DADBC_BUILD_SHARED=OFF \
        -DADBC_BUILD_STATIC=ON \
        -DADBC_BUILD_TESTS=OFF \
        -DADBC_BUILD_BENCHMARKS=OFF \
        -DADBC_BUILD_EXAMPLES=OFF \
        -DCMAKE_INSTALL_PREFIX=$TP_INSTALL_DIR \
        -DCMAKE_PREFIX_PATH="${TP_INSTALL_DIR}" \
        -DArrow_DIR="${TP_INSTALL_DIR}/lib64/cmake/Arrow" \
        -DArrowFlight_DIR="${TP_INSTALL_DIR}/lib64/cmake/ArrowFlight" \
        -DArrowFlightSQL_DIR="${TP_INSTALL_DIR}/lib64/cmake/ArrowFlightSQL" \
        -DCMAKE_BUILD_TYPE=Release \
        -G "${CMAKE_GENERATOR}" ..
    ${BUILD_SYSTEM} -j$PARALLEL
    ${BUILD_SYSTEM} install
}
```

Then add a call to `build_adbc` in the main build sequence, after the `build_arrow` call. Find the section in build-thirdparty.sh where `build_arrow` is called (search for `build_arrow$` or `build_arrow;`) and add `build_adbc` directly after it.

NOTE: The `-DArrow_DIR`, `-DArrowFlight_DIR`, `-DArrowFlightSQL_DIR` explicit paths are the recommended fallback from the research (pitfall 4). Use them to ensure cmake finds the already-built Arrow in $TP_INSTALL_DIR without relying on cmake auto-discovery.
