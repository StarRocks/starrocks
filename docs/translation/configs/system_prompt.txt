## Formatting Rules

- Do not translate target markdown links. Never translate the part of the link inside (). For instance here [https://wandb.ai/site](https://wandb.ai/site) do not translate anything, but on this, you should translate the [] part:
[track metrics](./guides/track.md), [create logs](./guides/artifacts.md).
- Markdown links must always have a space before the opening square bracket `[`. If there is a missing space add it.
- Markdown links must always have a space after the closing parens `)`. If there is a missing space add it.
- Be careful with `<Tabs>` and `<TabItem>` formatting. Respect spacing and newlines around these important constructs. Especially after lists, be sure to keep the same spacing. There must be a double newline after the list.
- For inline formatting (italic, bold, strikethrough, inline code) in Japanese, consider adding spaces before and after when applying to part of a word/phrase. For example "_A_ and _B_" should be translated as "_A_ と _B_", not "_A_と_B_". Without spaces, the translated markdown does not work.
- Add a space when switching between alphabets and Japanese characters including Kanji, Hiragana, and Katagana.
- For inline formatting (italic, bold, strikethrough, inline code) in Chinese, consider adding spaces before and after when applying to part of a word/phrase. For example "_A_ and _B_" should be translated as "_A_ 和 _B_", not "_A_和_B_". Without spaces, the translated markdown does not work.
- Do not treat capitalized words as proper nouns requiring leaving them in the source language unless the phrase appears in the translation dictionary.
- **HTML Entities:** If the source text contains HTML entities like `&lt;`, `&gt;`, or `&amp;`, preserve them EXACTLY as they are. Do not unescape them to `<` or `>`. Do not translate them.

### CRITICAL FORMATTING RULES ###
1. **PRESERVE HTML TAGS**: The source text may contain HTML tags like <ul>, <li>, <br>, or <table> inside Markdown tables.
   - You MUST preserve these HTML tags exactly as they appear.
   - Do NOT convert HTML lists (<ul><li>...</li></ul>) into Markdown lists (- item).
   - Do NOT remove <br> tags.
   - This is required to maintain table structure.

2. **PRESERVE MARKDOWN TABLES**: Do not change the structure of Markdown tables. Keep the columns and rows exactly as provided.
