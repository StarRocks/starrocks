## VARCHAR 型の最大長はどれくらいで、長さがクエリパフォーマンスにどのように影響しますか？

VARCHAR 型の最大長は 65533 で、1 MB のストレージサイズが必要です。VARCHAR の長さは必要最低限の値に設定することを推奨します。これは、VARCHAR 型のデータサイズは実際の長さに基づいているにもかかわらず、メモリの事前割り当てが必要なクエリシナリオでは、メモリリソースが実際の長さではなく、VARCHAR 型の事前定義された長さに基づいて割り当てられるためです。例えば、`address` フィールドには 100 バイトで十分であり、VARCHAR(100) が STRING よりも推奨されます。なぜなら、STRING 型は VARCHAR(65533) と同等だからです。
